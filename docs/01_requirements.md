# 需求文档

## 项目背景

我们有一批网络拓扑图数据（几百张 JSON 文件），每张图描述一个 AP 无线网络的完整配置。数据的核心是 `nodes`（设备节点）和 `deviceGroups`（设备组），其中 `configs[]` 携带配置信息。

数据格式详见 [05_data_format.md](./05_data_format.md)。

### 当前痛点

1. **配置很长**：单张图的上下文可达 100K+ tokens，直接阅读原始 JSON 效率极低。
2. **对配置不了解**：配置属于网络工程领域知识，不知道各类配置的含义、分布和规律。
3. **缺乏全局视角**：不清楚配置和拓扑之间存在什么关联。

---

## 项目目标

### 最终目标

研究 `deviceGroups` 和 `nodes` 中的 `configs` 配置信息，挖掘有价值的**配置预测下游任务**——利用拓扑信息（节点、连线、拓扑角色）来推理和预测配置。

### 实现路径

```
可视化 → 降低理解门槛 → 发现规律 → 定义有价值的预测任务
```

- **阶段一（当前）**：单图深钻 — 拓扑图 + 点击节点查看 configs
- **阶段二（后续）**：跨图统计 — 配置类型分布、节点-配置关联模式
- **阶段三（远期）**：规律驱动任务定义 — 基于发现的模式设计预测任务

---

## 阶段一需求：单图拓扑可视化

### 功能描述

选择一张网络拓扑图，以**交互式拓扑图**的形式展示：

1. **拓扑视图**：以 nodes 为节点、links 为边，渲染网络拓扑结构图。
2. **节点信息**：点击某个节点后，展示该节点的详细信息（设备属性、拓扑角色、configs 配置列表）。
3. **配置展示**：节点的 `configs[]` 以结构化、可折叠的方式呈现，支持按配置 key 展开/收起。
4. **设备组展示**：`deviceGroups` 以侧面板或列表形式展示其配置信息。
5. **图切换**：支持在 train/val 数据集中按文件名切换查看不同图。

### 用户场景

- 打开一张图 → 看到整体拓扑布局
- 点击一个节点 → 右侧面板显示该节点的 device 属性 + topologyNode 角色 + configs 列表
- 展开某段 config → 查看嵌套的配置参数
- 对比两个相邻节点的 configs → 发现同类设备是否有相似的配置

### 界面布局建议

```
┌──────────────────────────────┬──────────────────────┐
│                              │   节点详情面板          │
│                              │   - device 属性       │
│    拓扑图（力导向布局）        │   - topologyNode 角色  │
│                              │   - configs 列表       │
│    ● ← 节点                  │     ├ cloud-ap-ifaces  │
│    ─ ← 连线                  │     ├ ap-psk           │
│                              │     └ ...              │
│                              │                       │
├──────────────────────────────┼──────────────────────┤
│  工具栏：图选择器 | 布局切换   │  deviceGroups 面板     │
└──────────────────────────────┴──────────────────────┘
```

## 阶段一前置：全局统计（已完成）

在进入可视化之前，已完成一批全局统计脚本，建立对数据的初步认识：

| 脚本 | 统计维度 |
|------|----------|
| `node_field_stats.py` | 每图 nodes 数量、id/device/topologyNode/configs 字段覆盖率 |
| `node_subkey_stats.py` | device/topologyNode/configs 内部顶层 key 的分布和覆盖率 |
| `node_value_distribution.py` | TYPE/NET_ENVIRONMENT/NODECLASS/DEVICEROLE/CLASSNAME 的值分布 |
| `edge_type_pair_stats.py` | links 中边两端节点的 TYPE 配对统计 |
| `type_configs_stats.py` | 按 device.TYPE 分组统计 configs 配置类型分布 |

## 阶段三需求（规划中）：预测任务挖掘

（待对数据有充分理解后细化）

参考 `build_config_generation_dataset.py` 的"遮挡-预测"思路，结合可视化分析中发现的规律，可能的下游任务包括：

| 任务方向 | 描述 | 利用的拓扑信息 |
|----------|------|---------------|
| 单 key 预测 | 遮挡一个配置 key，用整图上下文预测 | 全局拓扑 + 邻居配置 |
| 新节点配置生成 | 拓扑中新增节点，生成应有的 configs | 邻居节点配置 + 拓扑角色 |
| 配置一致性检查 | 发现配置与同类节点差异大的异常 | 同角色节点对比 |
| 节点角色分类 | 根据 configs 推断 DEVICEROLE | configs → role 映射 |
| 配置补齐 | 部分 configs 缺失时补全 | 同角色节点完整配置 |

## 非功能需求

- **性能**：单张图渲染需在浏览器中流畅交互（力导向图 100+ 节点不卡顿）。
- **可扩展**：可视化组件需支持后续接入跨图统计视图。
- **数据兼容**：需兼容 `nodes[].config` 和 `nodes[].configs` 两种字段名，处理字段缺失的容错。
